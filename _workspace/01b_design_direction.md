## 0. 컨셉 한 줄 + 무드

**컨셉:** "쉼 없이 달리는 궤적 — 숫자로 멈춰 세우고, 진정성으로 붙잡는다."
다크 베이스 위 한 줄기 **라임-그린 트레이서(궤적선)**가 페이지를 관통하며 '쉼 없이 달리는 열정'을 시각화한다. 화려함이 아니라 *정돈된 속도감*.

**무드 키워드:** `정돈된 열정` · `속도/궤적` · `진정성` · `엔지니어 신뢰` · `증거 우선`

**안티무드(피한다):** 무지개 팔레트, 균일 3열 부트스트랩 카드, 스톡 일러스트/이모지로 빈칸 메우기, "제목+부제+버튼" 정형 히어로, 회색조만으로 끝내기. (브리프 안티-제네릭 체크리스트 전부 fail 처리 금지)

## 1. 컬러 토큰 (다크 베이스 택1)

다크 베이스를 택한다. 이유: ① 엔지니어 신뢰감(터미널·코드 정서), ② 라임 액센트와 숫자가 가장 강하게 튄다, ③ 멘토가 폰으로 밤에 열어도 눈부심 없음.

### Tailwind `theme.extend.colors` 그대로 복사

```js
colors: {
  // 베이스 (딥 차콜-네이비)
  ink:        '#0B0E14',   // 페이지 최하단 배경 (body)
  surface:    '#121620',   // 카드/섹션 박스 배경
  surface2:   '#1A1F2B',   // 카드 hover / 한 단계 띄운 면
  line:       '#262D3A',   // 디바이더·보더 (1px)

  // 텍스트
  fg:         '#EAEEF5',   // 본문 기본 (오프화이트, 순백 금지)
  'fg-muted': '#9AA4B6',   // 보조/캡션
  'fg-dim':   '#5E687A',   // 비활성·메타

  // 액센트 1 — 라임 그린 (메인 에너지/궤적/CTA)
  lime:       '#C6F24E',   // 핵심 액센트. 숫자·CTA·트레이서선
  'lime-700': '#9BCB1E',   // hover/active 딥 톤
  'lime-glow':'#C6F24E33', // 그림자/글로우용 (알파 20%)

  // 액센트 2 — 일렉트릭 시안 (보조, AI/기술 신호 한정)
  cyan:       '#5BE3D6',   // 아이디어 섹션 AI 스텝·기술 태그에만 절제 사용
}
```

**사용 규칙(엄수):**
- **액센트 2색 원칙.** lime = 에너지/전환/궤적 주역. cyan = AI·기술 맥락에서만 보조로(아이디어 섹션 AI 스텝, 허원일 카드의 기술 태그). 두 색을 한 요소에 섞지 말 것.
- 무지개·다색 그라데이션 금지. 그라데이션은 **히어로 배경 글로우 1곳**만 — `radial-gradient(circle at 70% 20%, #C6F24E14, transparent 60%)` 정도로 거의 안 보이게.
- 숫자(1,100+ / 1년 / 1,200+)는 항상 `lime`. 본문 텍스트에 lime을 흘리지 말 것 — lime은 '멈춤 신호'라 남발하면 가치가 떨어진다.

## 2. 타이포 시스템

폰트 패밀리 2개. 한글 가독성 본문 + 숫자/디스플레이용.

```
본문/한글:  Pretendard  (CDN: pretendard, 한글 가독성 최상·굵기 9단계)
디스플레이/숫자:  '디스플레이는 Pretendard Bold로도 가능하나,
                숫자 임팩트는 Space Grotesk(영문/숫자)로 변주'
```

**폰트 로딩(빌더):**
- `Pretendard` — `@import` 또는 `next/font` 로 variable. 전역 `font-sans`.
- `Space Grotesk` (Google Fonts) — 숫자·영문 라벨(MAU, MENTOR, 17기 등 영문 메타) 전용. `font-display` 유틸로 분리. 한글에는 절대 적용 금지(한글 글리프 깨짐).
- `break-keep`(한글 단어 단위 줄바꿈) 전역 적용. 헤드라인은 `<br>` 의도 지점 + `text-balance`.

**타입 스케일 (clamp 반응형, 375px~1440px):**

| 토큰 | clamp | 굵기/용도 |
|---|---|---|
| `display`  | `clamp(2.5rem, 7vw, 5.5rem)` | 히어로 헤드라인. Pretendard 800. line-height 1.08, letter-spacing -0.03em |
| `stat`     | `clamp(3rem, 9vw, 7rem)`      | **숫자 전용**(1,100+). Space Grotesk 700, lime. 페이지 최대 글자 |
| `h2`       | `clamp(1.75rem, 4vw, 3rem)`   | 섹션 헤더. Pretendard 700, -0.02em |
| `h3`       | `clamp(1.15rem, 2.2vw, 1.5rem)` | 카드 제목·대표증거. Pretendard 700 |
| `lead`     | `clamp(1.05rem, 1.6vw, 1.35rem)` | 히어로 서브·강조 본문. Pretendard 500, fg-muted |
| `body`     | `1rem / 1.05rem`              | 본문. Pretendard 400, line-height 1.75 |
| `caption`  | `0.8125rem`                   | 칩·메타. Space Grotesk 500 또는 Pretendard 500, tracking +0.04em, uppercase(영문만) |

**위계 원칙:** 임팩트는 `stat`(숫자)와 히어로 `display`에 몰아준다. 본문은 차분하게. 크기 점프를 분명히 — `lead`→`body` 사이에 어중간한 중간 사이즈 만들지 말 것.

## 3. 레이아웃 원칙

- **그리드:** max-width `72rem`(1152px) 중앙, 좌우 패딩 `clamp(1.25rem, 5vw, 2rem)`. 단, 히어로와 성장스토리는 의도적으로 비대칭(아래 아트디렉션 참조).
- **수직 리듬:** 섹션 간 `py-24 md:py-36`. 넉넉한 여백이 호흡 = 신뢰. 빽빽함 금지.
- **섹션 구분:** 배경색 교차로 끊지 말고(밋밋), `line` 색 1px 디바이더 + 좌측에서 시작하는 짧은 lime 궤적 마커로 구분.
- **모든 섹션 중앙정렬 금지.** 히어로·성장스토리는 좌측 정렬 비대칭, 팀/CTA는 중앙. 정렬 변주로 단조로움을 깬다.
- 카드 라운드 `rounded-2xl`(16px), 보더 `1px solid line`, 배경 `surface`. 그림자 대신 보더+미세 글로우로 다크 위에서 띄운다.

## 4. 시그니처 요소 (이 페이지만의 차별점)

### ★ 시그니처 1 — "러닝 트레이서(궤적선)"
'숨벅찬 = 쉼 없이 달리는' 을 직역한 **세로로 흐르는 라임 궤적선**. 히어로 하단에서 출발해 각 섹션의 좌측 마커를 잇고 CTA에서 결승선처럼 매듭짓는다.
- 구현: 좌측 고정 또는 섹션별 마커. 각 섹션 헤더 좌측에 `2px` lime 수직선 + 시작점 도트(`lime`, glow). 스크롤 진입 시 그 구간 선이 위→아래로 `scaleY(0→1)` 그려진다(SVG path stroke-dashoffset 또는 CSS).
- 모바일: 좌측 여백 좁으므로 선은 `12px` 들여쓰기, 도트만 유지. 깨지면 도트만 남기고 선 생략 가능.

### ★ 시그니처 2 — "오버사이즈 숫자 카운트업"
히어로의 `1,100+`, `1년`을 화면 최대 글자(`stat` 스케일, Space Grotesk)로 박고, 진입 시 **카운트업 애니메이션**(0→1,100). 숫자가 멘토의 스크롤을 멈추는 주역. 팀 섹션 각 카드의 대표 지표(1,200+ 등)도 같은 처리로 통일.

## 5. 모션 가이드

> 절제 원칙: 진입 페이드업 + 마이크로 인터랙션만. 과한 패럴랙스·자동 캐러셀 금지(신뢰↓, 모바일 깨짐).

| 모션 | 트리거 | 스펙 |
|---|---|---|
| 섹션 진입 | IntersectionObserver, 1회(`once`) | `opacity 0→1`, `translateY 16px→0`, `duration 600ms`, `ease-[cubic-bezier(0.22,1,0.36,1)]` |
| 순차 stagger | 카드/리스트 자식 | 자식마다 `delay +90ms` |
| 숫자 카운트업 | 진입 시 | `1100ms`, ease-out. `prefers-reduced-motion`이면 최종값 즉시 표기 |
| 트레이서선 | 해당 섹션 진입 | `scaleY/stroke-dashoffset` 그려짐, `700ms` ease-out |
| 버튼 hover | hover/focus | 배경 `lime→lime-700`, `translateY -2px`, `box-shadow 0 8px 24px lime-glow`, `150ms` |
| 카드 hover | hover | `surface→surface2`, 보더 `line→lime/40`, `200ms` |
| CTA 펄스 | idle, 절제 | 클로징 CTA 도트만 약한 glow 펄스(2.5s loop). 버튼 자체는 펄스 금지 |

**필수:** 모든 진입/카운트업/펄스는 `@media (prefers-reduced-motion: reduce)`에서 비활성(최종 상태 즉시). 빌더는 이걸 전역 가드로.

## 6. 섹션별 아트디렉션

> 강조 강도: **① 히어로(최강) > ⑤ CTA > ② 팀 > ③ 아이디어 > ④ 성장스토리**. 임팩트 예산을 이 순서로 배분.

### ① 히어로 — 최우선 강조 (7초 후크)
- **레이아웃:** 좌측 정렬 비대칭. 풀-블리드 다크(`ink`) + 우상단 거의 안 보이는 lime radial glow 1개. 중앙정렬 금지.
- **위계:** ㉠ 상단 작은 메타칩 `AI SW 마에스트로 17기 · 숨벅찬 청년들`(caption, Space Grotesk 영문+Pretendard). ㉡ `display` 헤드라인 — "이미 1,100명이 쓰는…" 에서 **`1,100`만 lime + stat급으로 인라인 점프**시켜 한 줄 안에서도 숫자가 튀게. ㉢ `lead` 서브카피(fg-muted, '숨벅찬 청년들'만 fg+700). ㉣ 하단 **증거칩 3개** 가로 배열(모바일 세로).
- **증거칩:** `실서비스 MAU 1,100+` / `2025 강원 AI 아이디어 경진대회 대상` / `3인 전원 풀몰입`. pill 형태 `border line`, 배경 `surface`, 좌측에 lime 도트. 숫자(1,100+)는 lime. 이게 멘토 3공포 선제해소 — 스크롤 없이 박히게 first-view 안에 배치.
- **시그니처 적용:** 트레이서선 출발점(상단 도트), 1,100 카운트업.
- **모바일(375):** 헤드라인 `break-keep`+clamp 하한 2.5rem, 칩 세로 스택 풀폭.

### ② 팀 소개 — 3카드 동등 비중
- **레이아웃:** 3열 그리드(`md:grid-cols-3`, 모바일 1열). **단, 부트스트랩 균일 카드 금지** — 각 카드를 개성화:
  - 좌측에 멤버 이니셜/역할 인덱스를 Space Grotesk 대형(`01 BE` / `02 FE·INFRA·AI` / `03 FULLSTACK`)로 박아 인물 구분.
  - **대표 증거 한 줄을 `h3` 굵기로 카드 최상단**(브리프 의도: 가장 굵게). 권상재=배포·CI/CD, 허원일=AI 자동화, 황선규=MAU 1,100+.
  - 황선규 카드의 `1,200+`/`1,100+`, 지표는 lime + 카운트업(시그니처2 재사용). 허원일 카드 기술 태그(Claude Code·MCP·K8s)는 cyan 보더 칩으로 AI 신호.
- **프로필 링크(★갱신):** 각 카드 하단에 채널 링크 줄 — `프로필 더보기(Notion)` · `GitHub` · (허원일만)`velog` · `오픈채팅`. 텍스트 링크 칩(caption, fg-muted, hover 시 lime + 밑줄), 외부링크 `target=_blank rel=noopener`, `aria-label`에 "(이름) (채널)". URL: USER_DECISIONS.md Notion 링크 + digest의 GitHub/velog/오픈채팅. 전화·이메일 직접 노출 금지.
- **헤더:** "한 명도 빠진 역할이 없습니다." h2. 케미 한 줄은 lead/muted로 헤더 하단.
- **'거리를 열정으로 좁힌 팀' 보조라인(★갱신):** 케미 한 줄 바로 아래 작은 텍스트(caption/dim)로 — 권상재 통근 불사(센터까지 1시간), 황선규·허원일 부산 센터 인근 자취. '풀몰입·시간 투입' 신호를 인간적으로 강화. 좌측에 작은 lime 핀/도트 아이콘 1개로 위치 뉘앙스만, 과장 금지.
- **개성 장치:** 카드 hover 시 해당 역할 인덱스 숫자가 lime으로 점등 → "퍼즐이 맞물린다"는 느낌.

### ③ 아이디어 — 인터랙티브 마인드맵 (중앙 허브 + 방사형 후보 노드) ★갱신

> **변경 이력(2026-06-01):** 사용자 확정으로 정적 3스텝 플로우 → **인터랙티브 마인드맵**으로 전면 교체. 중앙 허브 1 + 가지 노드 5(아이템) + 빈 노드 1(멘토 초대) = 총 6 노드. 노드 클릭/탭 → 상세 패널. 프레이밍 유지: 후보는 '확정'이 아니라 "검토 중·멘토와 함께 고를 거리". 시그니처 '러닝 트레이서'를 엣지(가지)로 녹인다. 다른 섹션 토큰 불변.

- **헤더:** "무엇을 만들지, 함께 고를 준비를 해왔습니다 / **고르는 건 멘토님과 함께.**" h2, 후반부만 lime under-stroke. 도입 카피(`body`/muted)에서 'AI를 실제로 쓰는, 사람이 돈을 낼 만한 서비스'만 fg+600, 'AI' 토큰은 cyan. "검토 중인 출발점" 한 줄은 caption/dim으로 마인드맵 위에 라벨.

#### 레이아웃 — 방사형 마인드맵
- **데스크톱(≥768px):** 중앙 허브 노드를 캔버스 중심에. 가지 노드 5개를 **방사형(약 -90°~+270°를 5등분, 황금각 분산 권장)**으로 배치 — 균일 원형 배열은 '시계 같다'는 인상이라 반지름을 노드마다 8~16% 흔들어(`radius * (0.92~1.08)`) 유기적으로. 빈 노드 ⑥은 가지들 바깥 빈 섹터에 살짝 떨어뜨려 '아직 안 채워진 자리'로 보이게.
- **캔버스 크기:** `min-h-[clamp(520px,70vh,720px)]`. 노드가 캔버스 밖으로 안 나가게 좌표는 % 기반 + `max-w` 클램프.
- **모바일(<768px):** 방사형 붕괴 → **세로 스택 리스트**로 우아하게. 상단에 허브 노드(풀폭 강조 카드), 아래로 가지 노드 6개를 세로 리스트(각 노드 = 탭 가능한 행, 좌측 lime/cyan 도트 + 라벨 + chevron). 방사 좌표 계산을 모바일에서 끄는 분기 필수. 가짜 SVG 선을 모바일에서 그리지 말 것(겹침·깨짐).

#### 엣지 = 러닝 트레이서 (시그니처 녹이기)
- 허브 → 각 가지 노드를 잇는 선은 **lime 트레이서 궤적**(SVG `<path>`, 약한 곡률 = `quadraticCurveTo`로 휘어 '뻗어나가는' 느낌). 직선 금지(딱딱함).
- **그려짐 모션:** 섹션 진입 시 허브에서 각 가지로 `stroke-dashoffset` 애니메이션이 **순차 stagger(+90ms)**로 뻗어나간다 — 700ms ease-out. 이게 "중앙에서 에너지가 퍼지는" 핵심 인상.
- 선택된 노드의 엣지는 굵기 `1.5px→2.5px` + lime glow 강조, 비선택 엣지는 `line` 색으로 디밍.
- 빈 노드 ⑥의 엣지는 **점선(`stroke-dasharray`) + cyan** — '미완성·초대' 신호를 선에서부터.

#### 노드 상태 컬러 (라임/시안 2액센트로 구분)
| 상태 | 가지 노드(①~⑤) | 빈 노드(⑥) |
|---|---|---|
| 기본 | 배경 `surface`, 보더 `line`, 라벨 `fg`, 좌측 도트 `lime` | 배경 transparent, **점선 보더 cyan/50**, 라벨 `fg-muted`, `+` 아이콘 cyan |
| 호버/포커스 | 보더 `lime/50`, 배경 `surface2`, `translateY -2px`, 미세 lime glow | 점선 보더 `cyan`, 배경 `cyan/8`, `+` 회전 90°(45ms) 마이크로 |
| 선택(active) | 보더·도트 `lime` 솔리드, 배경 `surface2`, 엣지 강조 | 보더 `cyan` 솔리드, 라벨 fg |
- 노드 라벨은 짧게(브리프 라벨 그대로: `계약서·문서 검토 자동화` 등). 라벨 길면 2줄 `break-keep`. 노드는 pill/rounded-xl 카드, 터치타겟 ≥44px.

#### 상세 펼침 — 패널
- **데스크톱:** 우측 **사이드 패널**(`w-[380px]`, 캔버스 우측에 슬라이드 인, `translateX 16px→0` + fade, 300ms). 마인드맵은 좌측에 남아 선택 노드가 lime로 점등 → 맥락 유지. (모달보다 패널 권장: 비교 탐색이 쉬움)
- **모바일:** **바텀시트 모달**(하단에서 올라옴, `translateY`, 배경 스크림 `ink/70`, 드래그/X/스크림탭으로 닫기). 풀스크린 모달은 과함 — 바텀시트로.
- **패널 내용(4축 통일):** 노드 라벨(h3) + `검토 중인 후보` 배지(caption, lime/cyan 보더) + **[문제 / 타깃 / AI 적용 / 수익 가설]** 4행. 각 축 라벨은 caption/dim, 내용은 `body`. AI 적용 행의 강점 매칭(예: "허원일 MCP 경험 직결")은 cyan 미세 텍스트로 — 멘토에게 '증거 연결' 신호.
- **빈 노드 ⑥ 패널:** 4축 대신 "멘토님이 보시는 시장 기회를 여기 함께 채우고 싶습니다" 한 카피 + `+ 함께 그릴 후보` 대형 + cyan. 입력폼 만들지 말 것(정적 페이지) — '초대' 메시지로만.

#### 접근성 (필수)
- 노드 = `<button>`(또는 `role="button"` + `tabindex=0`). `aria-label`에 노드 라벨, `aria-expanded`로 패널 열림 상태, 패널은 `role="dialog"`(모바일 모달) 또는 `aria-live` 영역(데스크 패널).
- **키보드:** Tab으로 노드 순회(시각적 방사 순서 ≠ DOM 순서일 수 있으니 DOM은 ①~⑥ 논리 순서로), Enter/Space로 펼침, Esc로 패널 닫기, 포커스 트랩(모바일 모달)·닫으면 트리거 노드로 포커스 복귀.
- `prefers-reduced-motion`: 엣지 그려짐·패널 슬라이드 비활성 → 즉시 표시.

#### 구현 난이도 가이드 (빌더 판단용)
- **권장(가벼움):** 외부 그래프 라이브러리(react-flow 등) **불필요**. 노드 좌표는 **React state + absolute positioning(% 좌표)**, 엣지는 **단일 `<svg>` 오버레이에 `<path>` 6개**(허브 중심→노드 중심 좌표로 계산). 선택 상태는 `useState<selectedId>`. 의존성 0개로 충분.
- 좌표 계산: 노드 6개 각도/반지름을 상수 배열로 두고 `cx + r*cos(θ)` 산출. 캔버스 리사이즈는 `%`라 자동 대응.
- 라이브러리를 쓰고 싶다면 react-flow도 가능하나 **번들·모바일 무게 트레이드오프**가 있으니, 노드 6개 규모에선 자체 구현을 1순위로 제안. (빌더가 최종 판단)
- 모바일은 위 세로 스택이라 SVG 좌표 로직을 탈 필요 없음 — 데스크톱 캔버스와 모바일 리스트를 **별도 분기 렌더**(같은 데이터, 다른 뷰)로 두면 구현·디버깅이 쉽다.

### ④ 성장 스토리 — 타임라인/인용구 중심
- **레이아웃:** 좌측 정렬 **세로 타임라인**(트레이서선 본체가 여기서 가장 길게 흐름). 노드 3개 = 3인의 태도 증거.
- **인용구 우선(브리프: 숫자보다 태도):** 각 노드를 **풀-너비 인용구**로. 허원일 "배포에서 끝내지 않고 사용자 피드백 반영·개선까지" / 황선규 "실패는 성장의 과정" / 권상재 "모르는 것 숨기지 않고 빠르게 공유". 큰따옴표 글리프를 lime 대형 장식으로, 인용은 `lead` 굵기.
- **착지 문단:** "멘토님의 피드백은 버려지지 않습니다" → CTA로 잇는 다리. `버려지지 않습니다` lime 강조. 타임라인 종착 도트가 다음 섹션(CTA) 결승선으로 연결되게.
- **숫자 절제:** 이 섹션엔 stat 숫자 쓰지 말 것 — 톤·진정성 담당이라 숫자 임팩트를 빼서 다른 섹션과 차별.

### ⑤ 멘토에게 / CTA — 두 번째 강조, 결승선
- **레이아웃:** 중앙정렬, 페이지에서 **두 번째로 강한 시각 무게**. 배경을 `surface`로 한 단 띄우고 트레이서선이 여기서 매듭(결승 리본 모티프 — 좌우로 짧게 퍼지는 lime 라인).
- **위계:** ㉠ 헤더 "기본기·운영·AI는 우리가 채웠습니다…" ㉡ 본문(muted) ㉢ **클로징 "멘토님이 쓰신 시간은, 다음 주 결과물로 돌려드립니다."** — `h2`급 + `결과물`만 lime. 페이지에서 히어로 다음으로 굵은 카피. ㉣ CTA 버튼.
- **CTA 구조(★갱신 — 팀 대표 이메일 제거):** 사용자 확정으로 팀 대표 이메일/단일 채널 없음. 단일 메일 버튼 대신 **멤버 3인 개별 프로필 칩**으로 유도.
  - ㉠ 헤드 카피 위 `함께 만들기 →` 는 **앵커 버튼**(클릭 시 아래 멤버 칩 영역으로 스크롤/포커스)으로 유지 — lime 솔리드 + ink 텍스트, `font-display` 화살표, hover -2px+glow. 시각적 '주 버튼'은 1개로 유지(과밀 방지).
  - ㉡ 그 아래 **멤버 프로필 칩 3개**(권상재/허원일/황선규) 가로 배열(모바일 세로). 각 칩 = 이름 + 역할 한 단어 + 채널 아이콘 링크(`프로필(Notion)` / `GitHub` / `오픈채팅`, 허원일은 `velog` 추가). ② 팀 카드의 링크와 **동일 URL 재사용**(USER_DECISIONS.md Notion 링크 + digest의 GitHub/velog/오픈채팅). 전화·이메일 직접 노출 금지.
  - 칩 스타일: `border line`, hover 시 보더 lime/50 + 채널 아이콘 lime 점등. 터치타겟 ≥44px, 외부링크는 `target=_blank rel=noopener` + `aria-label`에 "(이름) (채널)".
- **CTA 도트 펄스:** 결승 도트만 약한 glow 펄스(절제). 버튼 자체는 펄스 금지.
- **모바일:** 주 버튼·멤버 칩 풀폭, 터치타겟 ≥52px. 화면 하단 sticky CTA는 선택(빌더 판단) — 단일 채널이 없으므로 sticky는 '연락' 앵커로만, 본문 가림 주의.
